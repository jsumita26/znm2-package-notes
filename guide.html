# 1 个 GitHub 账号 + 5 个仓库 + 5 个真实内容页 下面这套方案的目标是： * 每个域名都有不同主题 * 每个页面都足够轻量 * 内容与你现有兴趣一致 * 适合直接放到 GitHub Pages ## 推荐的 5 个站点定位 ### 1. 编译总流程站 * 仓库名：`znm2-build-flow` * 域名：`build.example.sld.tw` * 主题：ZN-M2 编译固件整体流程 * 页面关键词：环境准备、feeds、menuconfig、编译、刷写前检查 ### 2. 选包与精简站 * 仓库名：`znm2-package-notes` * 域名：`pkg.example.sld.tw` * 主题：哪些包该留，哪些可删 * 页面关键词：1G 内存、不要 USB、不要 WiFi、ZeroTier、多 WAN ### 3. 代理与网络功能站 * 仓库名：`znm2-network-lab` * 域名：`net.example.sld.tw` * 主题：ZeroTier / 多 WAN / 代理功能实验记录 * 页面关键词：功能拆分测试、依赖冲突、轻量优先 ### 4. 刷机与排错站 * 仓库名：`znm2-flash-notes` * 域名：`flash.example.sld.tw` * 主题：刷机前检查与故障排查 * 页面关键词：factory、sysupgrade、回退方案、日志、串口 ### 5. 更新日志与索引站 * 仓库名：`znm2-lab-index` * 域名：`lab.example.sld.tw` * 主题：你的总导航页 * 页面关键词：项目入口、更新时间、各站点链接 --- ## GitHub Pages 部署方式 每个仓库都放一个 `index.html` 即可。 然后在每个仓库里打开： * Settings * Pages * Deploy from a branch * Branch 选 `main` * Folder 选 `/ (root)` * Custom domain 填对应的 `xxx.sld.tw` DNS 侧： * 每个域名只添加 1 条 `CNAME` * 记录值指向：`你的GitHub用户名.github.io` --- ## 首页模板 1：编译总流程站 文件名：`index.html` ```html ZN-M2 编译流程
